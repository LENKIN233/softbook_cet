01Pillars

五根承重柱，其余都可以换。

Law of One

一屏一个当前学科，一屏一种强色。

用户一次只在一个 library 里（听力蓝 / 阅读橙 / 写作紫 ...）。当前学科色是这屏的主轴，所有 chip、picked、分隔、重点高亮都向它靠拢；其余 6 色只能以 10–20% 透明度出现在导航或空间地图里。违反即失格。

Pillar 01

Aurora 氛围，不是装饰

背景的 3–5 个色球是当前屏幕情绪的暗示——玻璃之所以显色，是因为身后真有色。不要在玻璃上再叠装饰。

← interface_principles.quiet + glass_depends_on_atmosphere
Pillar 02

玻璃有厚度

40–60px 模糊 + 1px 亮边 + inset 顶光 + 4% grain。纸是克制的玻璃，玻璃是克制的体积。不做平面色块。

← card-as-object · VisionOS vocabulary
Pillar 03

排印承载权威

Outfit display 驱动标题（-0.045em 紧字距，800–900），Noto Sans 驱动中文正文，JetBrains Mono 驱动数字与 label。衬线不来，但权威感靠尺度与紧度建立。

← desk-authority reimagined in modern sans
Pillar 04

交互即轮廓

翻面 / 四选一 / 开锁 / 消除 / 滑动——五种交互的屏幕轮廓必须远看可辨。不是"四个按钮换图标"。

← interaction_contract_status · interactions.json
Pillar 05

空间是彩色地图

七学科色让 library 在空间视图里一眼可认。sleep 不是灰色，是"掉饱和 + 降透明度"。层级永远 library→group→box→card。

Night ≠ Inverted Day

  • 漆面 lacquer 而非纯黑。背景 #0B0B12，玻璃 rgba(28,28,42,.72)，文字 #F2F2FA——所有值都偏冷蓝微紫，不用纯黑纯白。
  • 学科色不变，但氛围球亮度 +20%。夜里玻璃更透，需要更亮的身后色把玻璃"点亮"。
  • 反馈色保持同色相，-10% 饱和。#22C58B → 看起来更沉，但可辨识。
  • 阴影关闭，改用 1px 亮边分层。夜里阴影不再承担分层——上亮边 + 下暗边解决。
  • 所有排印权重 -100。display 从 800 → 700，body 从 400 → 300。夜里眼睛不需要那么重的字。

07Canon

做 与 不做。两份清单，交付前照查。

做 DO

  • 一屏一个当前 library，一屏一种强色（学科色）。
  • 玻璃 40–60px 模糊 + 1px 亮边 + inset 顶光 + 4% grain。
  • 数字一律 tabular-nums + Outfit 800，统计不跳动。
  • 空间视图用七学科色直接识别，sleep = 掉饱和 + 降透明度。
  • 付费墙先解释"为什么此刻被拦"，再给价格，再给 CTA。
  • tabbar 用胶囊浮岛玻璃，不贴底全宽。
  • 图标 1.75 px stroke, rounded joins, 16–20px 尺寸。
  • 动效 240–360ms, cubic-bezier(.2,.8,.2,1)，全部能用物理解释。

不做 DON'T

  • 不做 streak 礼花 / 徽章 / XP / 等级 / 排行榜。
  • 不做渐变文字大标题，不做彩虹描边。
  • 不做"AI 思考中"的粒子或光效。
  • 不在单屏上让两种学科色同时作为强色出现。
  • 不用纯黑 #000 / 纯白 #FFF / 100% 饱和色做大块面积。
  • 不把 favorite 画成物理盒，不把 hint 做成独立卡型。
  • 不把空间缩成"收藏 + 休眠"两盒。
  • 不自动播放音频，不做波形跳动动画。
  • 不做趋势大盘、不做"本周洞察"卡片。
