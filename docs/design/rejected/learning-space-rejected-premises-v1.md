## Purpose

Rejected ideas are part of the design harness. They prevent future agents from rediscovering attractive but wrong paths.

## Rejected Premises

### 1. Dashboard Learning

Failure:

- makes the user manage a system instead of learning one current card;
- promotes counters, charts, and progress modules above exam content;
- violates the low-burden single-card flow.

Do not revive as:

- streak-first home;
- percentage dashboard;
- module control center;
- “today's report” as the primary Learning screen.

### 2. Flashcard Stack Only

Failure:

- collapses Softbook into an ordinary memorization tool;
- hides library / group / box / card address;
- cannot carry the physical-space differentiator.

Do not revive as:

- generic white card plus answer button;
- swipe-only deck;
- vocabulary-first app shell.

### 3. Freeform Knowledge Garden

Failure:

- asks the user to maintain a knowledge system;
- conflicts with system sequencing and partial user control;
- turns box ownership into user-authored organization.

Do not revive as:

- arbitrary drag-and-drop between boxes;
- user-created library taxonomy;
- canvas-style spatial editor.

### 4. Game Board

Failure:

- weakens exam trust;
- shifts motivation from passing CET to achievement collection;
- invites forbidden XP, badge, confetti, level, and medal chrome.

Do not revive as:

- level map;
- quest path;
- card battle metaphor;
- celebratory reward layer after ordinary answers.

### 5. Favorite Box

Failure:

- contradicts `favorites_role = tag`;
- reduces physical space to a collection shortcut;
- makes favorite look like a location change.

Do not revive as:

- “收藏盒” in Space hierarchy;
- moving cards into favorite as primary management action;
- favorite-only filter presented as a physical shelf.

### 6. Pure Lecture Page

Failure:

- earns exam trust but hides objecthood and spatial address;
- makes Space feel like a static table of contents;
- risks becoming a course or notes product.

Do not revive as:

- chapter reading page;
- long scrolling analysis as primary learning path;
- textbook screen without interaction silhouette.

### 7. Pure Visual Object Demo

Failure:

- makes material effects the product;
- harms content clarity;
- turns Aurora Glass into decoration instead of hierarchy.

Do not revive as:

- oversized glass object with small exam content;
- animated layer demo before answer clarity;
- map fragments that compete with the current card.

## Allowed Salvage

- Editorial authority from Pure Lecture Page may be used for typography.
- Fast action rhythm from Focus Console may be used for interaction clarity.
- Material depth from Knowledge Object may be used only when it explains current card, hint, or address.
