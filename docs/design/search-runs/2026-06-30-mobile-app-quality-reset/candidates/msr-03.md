# Space First Shelf

## Candidate ID

msr-03

## Provenance

- Tool or model: Codex design synthesis
- Prompt: Phone reset that makes the current card visibly pulled from a Space shelf while preserving Learning continuity.
- Source context pack: `context-pack.md`
- Artifact: `candidate-proofs/mobile-reset-candidate-proof.html#msr-03`
- Screenshots: `candidate-proofs/mobile-reset-candidate-proof.html#msr-03`

## Product Truth Fit

Strongly preserves Physical Space hierarchy and Learning to Space continuity. It risks weakening Learning as the top entry if shelf browsing becomes the first action.

## Focal Object

The current box shelf with the active card pulled forward.

## First-Read Path

Current box shelf -> active card -> primary action -> navigation capsule.

## Interaction Silhouette

Hybrid Learning plus Space hierarchy silhouette. It is acceptable only if the active card remains more important than browsing.

## Spatial Model

Library / group / box / card are visible as nested physical levels. Favorite is a tag on cards; sleep is a zone attached to the box, not a top-level box shortcut.

## State Language

Card states appear as small tags attached to objects. The shelf preserves current location before showing actions.

## Motion Causality

The card moves forward from its box because Learning selects it. Returning to Space moves the card back to its physical address.

## Platform Strategy

Phone shows a compressed shelf. Tablet and PC can expand the hierarchy, but phone must keep browsing secondary.

## Implementation Mapping

Maps primarily to Space address aperture plus Learning object plane. Future RN should not promote full shelf browsing on the Learning screen.

## Known Risks

Can become a browsing surface. It may visually over-explain Space and reduce the one-card action priority.

## Design Review Checklist Answers

Q1: Current library is Reading, with one coral active edge and muted hierarchy context.

Q2: Focal object is the active card pulled from the current box.

Q3: Space hierarchy silhouette is strong and Learning interaction remains attached to the active card.

Q4: It avoids forbidden patterns and does not require dashboard or full-width tab chrome.

Q5: Phone containment is risky but feasible if the shelf is compressed to one address band.

Q6: Learning must remain system-sequenced; this candidate cannot become a module picker.
