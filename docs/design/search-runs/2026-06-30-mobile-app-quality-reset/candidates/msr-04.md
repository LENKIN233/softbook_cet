# Action Console Minimal

## Candidate ID

msr-04

## Provenance

- Tool or model: Codex design synthesis
- Prompt: Phone app reset with one primary action surface and strict secondary action suppression.
- Source context pack: `context-pack.md`
- Artifact: `candidate-proofs/mobile-reset-candidate-proof.html#msr-04`
- Screenshots: `candidate-proofs/mobile-reset-candidate-proof.html#msr-04`

## Product Truth Fit

Preserves low operation cost and first-action clarity. It risks losing the distinctive physical-space and softbook material identity if the console becomes too utilitarian.

## Focal Object

The primary answer action area beneath the current card.

## First-Read Path

Question object -> answer control -> small Space address -> chrome.

## Interaction Silhouette

Strong for multiple choice and auto-scored interactions. Weaker for flip and Space continuity unless supported by attached layers.

## Spatial Model

Space is shown as a compact line of physical address. It is visible but not rich.

## State Language

Correctness and recovery are shown through the primary action area. Favorite and hint are icon-level secondary tools.

## Motion Causality

Motion follows answer selection: option confirmation collapses into result, then a small continuation layer appears.

## Platform Strategy

Phone maps cleanly because the action area is bounded. Tablet and PC can add richer Space context beside the action plane.

## Implementation Mapping

Maps cleanly to action plane logic in `LearningSurface`, but needs a separate Space implementation to preserve product differentiation.

## Known Risks

It can look like a generic quiz app. It may underplay CET trust and physical Space.

## Design Review Checklist Answers

Q1: Current library is Reading and the action accent uses only coral.

Q2: Focal object is the answer action area attached to the current card.

Q3: Interaction silhouette is clear for auto-scored Learning states, but Space silhouette is weak.

Q4: It avoids forbidden visual patterns and keeps chrome quiet.

Q5: Phone containment is strong because the action plane is bounded.

Q6: Learning remains sequenced; stats and flip-specific rules are unchanged.
