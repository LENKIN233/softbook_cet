# Fragment Harvest

## Source Run

`2026-06-30-mobile-app-quality-reset`, generation 1.

## Best Focal Object

`msr-01`: the current card object remains visually central across Learning, resolved detail, and Space continuation. It avoids turning the screen into a set of equal cards.

## Best First-Read Path

`msr-01`: current object -> primary action or result slip -> attached Space address -> floating app chrome. This is the clearest path for leadership screenshots and real use.

## Best State Language

`msr-02`: the answer slip and desk layer make resolved state feel like something attached to the same study object. The useful part is the material continuity, not the denser book metaphor.

## Best Space Or Interaction Model

`msr-03`: the visible address shelf proves library / group / box / card without making favorite a box. The winning synthesis should borrow this as a secondary layer inside `msr-01`.

## Best Motion Causality

`msr-01`: the result detail should slide from the current object because the user resolved the card; Space address should rise because the user asks where the card lives. Motion follows state cause, not decoration.

## Best Platform Adaptation

`msr-04`: the primary-action discipline maps cleanly to phone first and can expand on tablet and pc web without inventing new product semantics.

## Risky But Useful Breakthrough

`msr-03` makes Space more visible from Learning. Usefulness: it repairs the current app's weak Learning to Space continuity. Risk: Space can overtake Learning. Rollback condition: if the first-read path starts at browsing rather than the current card, demote Space back to an address aperture.

## Rejected Failure Patterns

- Timeline-first learning management from `msr-05`.
- Metrics-first dashboard composition from `msr-06`.
- Carousel-first spatial browsing from `msr-07`.
- Report-first answer detail from `msr-08`.

These failures are recorded in `docs/design/rejected/mobile-core-surface-reset-failures-v1.md`.

## Synthesis Inputs

Use `msr-01` as the base. Add `msr-02`'s answer-slip material, `msr-03`'s compact spatial address, and `msr-04`'s primary-action discipline. Do not carry over timeline, dashboard, carousel-first, or report-first structures.
