# Hard Filter Results

## Filter Scope

Generation 1 mobile app quality reset for phone. The applied filters were product truth, one-screen Learning flow, Law of One, Space physical hierarchy, result-detail continuity, Statistics quietness, Mine product fit, forbidden visual patterns, phone containment, and implementation-authority boundary.

## Rejected Candidates

- `msr-05`: rejected because the timeline becomes the main path and makes the user manage progress rather than operate the current card.
- `msr-06`: rejected because the dashboard surface competes with the current card and makes metrics feel like the product center.
- `msr-07`: rejected because the map carousel makes browsing the primary action and weakens the Learning next step.
- `msr-08`: rejected because result detail becomes the first-class page, preserving the report feeling that triggered the reset.

## Surviving Candidates

- `msr-01`: passed one-screen Learning, result-slip continuity, quiet chrome, Law of One, and Space continuity filters.
- `msr-02`: passed physical object continuity and CET trust filters, but carries higher material-density risk.
- `msr-03`: passed Space hierarchy and Learning to Space continuity filters, but weakens Learning as the primary entry.
- `msr-04`: passed primary-action clarity and phone containment filters, but risks becoming too utilitarian.

## Product Truth Violations

`msr-05` violates the system-sequenced single-card flow by foregrounding timeline management. `msr-06` violates low-burden learning by making dashboard metrics visually dominant. `msr-07` risks turning Space into browsing chrome. `msr-08` violates Learning state continuity by turning answer detail into a report page.

## Layout Or Proof Violations

No candidate failed to render in the shared proof file. `msr-06` and `msr-08` show too many equal-weight regions for phone. `msr-07` uses a wide carousel mental model that would require careful clipping and is not a safe reset foundation.

## Notes For Mutation

- Preserve `msr-01`'s current object rail and attached result slip.
- Borrow `msr-02`'s book-desk material only where it supports CET trust.
- Borrow `msr-03`'s visible Space address as a secondary layer.
- Borrow `msr-04`'s primary-action discipline without making the app feel like a utility console.
- Reject timeline, dashboard, carousel-first, and report-first structures for the next generation.
