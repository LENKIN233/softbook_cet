# Pairwise Review: msr-02 vs msr-03

## Pair

- Candidate A: `msr-02`
- Candidate B: `msr-03`

## Reviewer Role

Reviewer checks CET product fit, next-action clarity, app visual grammar, interaction state cause, physical Space continuity, implementation mapping, and phone layout containment.

## Winner

Candidate A, `msr-02`.

## Visual Evidence

- Candidate A evidence: `candidate-proofs/mobile-reset-candidate-proof.html#msr-02`
- Candidate B evidence: `candidate-proofs/mobile-reset-candidate-proof.html#msr-03`

## Product Truth

`msr-03` better proves Space hierarchy, but `msr-02` better protects Learning as the primary user entry. Because the reset is for the whole app shell, the winner cannot make browsing the first read.

## Task Clarity

`msr-02` keeps the user closer to the current card. `msr-03` makes the user parse shelf structure before acting.

## Space Or Interaction Fit

`msr-03` has the better Space model. `msr-02` has the better Learning continuity. The promoted synthesis should borrow `msr-03`'s address shelf without adopting its Space-first priority.

## Visual System Fit

Both can preserve Law of One. `msr-03` risks multiple colored regions in a complete Space view. `msr-02` has less accent conflict.

## Implementation Mapping

`msr-02` maps more cleanly to a Learning result state. `msr-03` maps well to Space but would be heavy as the general app shell.

## Rationale

Choose `msr-02` in this pair because the app's first promise is guided Learning. Harvest `msr-03`'s spatial address as a secondary layer.

## Borrowable Fragments

- From `msr-03`: compact address shelf and current box focus.
- From `msr-02`: answer slip continuity.

## Rejected Fragments

- From `msr-03`: full shelf as the first object in Learning.
