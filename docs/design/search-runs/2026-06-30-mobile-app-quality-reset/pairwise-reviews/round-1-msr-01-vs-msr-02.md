# Pairwise Review: msr-01 vs msr-02

## Pair

- Candidate A: `msr-01`
- Candidate B: `msr-02`

## Reviewer Role

Reviewer checks CET product fit, next-action clarity, app visual grammar, interaction state cause, physical Space continuity, implementation mapping, and phone layout containment.

## Winner

Candidate A, `msr-01`.

## Visual Evidence

- Candidate A evidence: `candidate-proofs/mobile-reset-candidate-proof.html#msr-01`
- Candidate B evidence: `candidate-proofs/mobile-reset-candidate-proof.html#msr-02`

## Product Truth

`msr-01` better preserves the current-card operation and Learning to Space continuity. `msr-02` preserves CET trust, but the book-desk metaphor can become a material style rather than an app interaction grammar.

## Task Clarity

`msr-01` makes the next action clearer because the answer slip and Space address attach to the current object. `msr-02` asks the user to read the material metaphor before acting.

## Space Or Interaction Fit

`msr-01` keeps Space as an address aperture while preserving Learning. `msr-02` uses a bookmark-like Space cue, which is useful but weaker.

## Visual System Fit

Both preserve Law of One. `msr-01` has the cleaner focal hierarchy. `msr-02` risks too many material surfaces.

## Implementation Mapping

`msr-01` maps directly to object plane, action plane, answer slip, and address aperture. `msr-02` would require more bespoke material layout.

## Rationale

Choose `msr-01` as the base and harvest `msr-02`'s answer-slip material. The product needs a coherent app grammar more than a stronger book metaphor.

## Borrowable Fragments

- From `msr-02`: folded answer slip, quieter explanation density, credible study-material feel.
- From `msr-01`: persistent current object and attached layer model.

## Rejected Fragments

- From `msr-02`: dense desk/page surfaces and any metaphor that makes Learning feel like a document reader.
