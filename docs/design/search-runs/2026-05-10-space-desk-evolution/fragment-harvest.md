# Space Desk Evolution Fragment Harvest

## Source Run

`docs/design/search-runs/2026-05-10-space-desk-evolution`, generation 1, Space phone surface.

## Best Focal Object

`space-04` has the strongest focal object: an open current box tray with one reading-coral edge and contained card objects. It is more concrete than the baseline because the box reads as a working surface instead of only a large card frame.

## Best First-Read Path

`space-04` wins first-read path: address shelf -> open box tray -> active card strip -> sleep alcove -> return to Learning. `space-03` contributes a clear side shelf, but it needs `space-04`'s desk plane to avoid feeling like a file browser.

## Best State Language

`space-05` has the best sleep/wake state language because sleeping cards sit in a quieter alcove under the same box label. `space-04` should borrow the alcove placement while keeping the current box as focal object.

## Best Space Or Interaction Model

`space-04` best preserves Space hierarchy: library/group context remains visible, the current box is first-read, cards remain contained, and state zones are attached to the box instead of becoming separate destinations.

## Best Motion Causality

`space-05` offers the cleanest motion cause: a card lowers into the alcove when slept and lifts back to the tray when woken. This causality should be described as physical state movement, not as animation decoration.

## Best Platform Adaptation

`space-03` adapts best to tablet and pc web because the shelf spine can expand into a left rail while the desk tray remains the object plane. The promoted synthesis should preserve that region split.

## Risky But Useful Breakthrough

`space-04` introduces a side shelf plus open desk tray. Usefulness: it makes the parent hierarchy legible without turning the screen into a module picker. Risk: if over-rendered, the shelf could become navigation-first. Rollback condition: if the current box is not first-read in a squint test, drop the shelf to a quieter address ladder.

## Rejected Failure Patterns

- Progress board first-read from `space-06`: sediment as a Space drift risk because it replaces physical hierarchy with management status.
- Favorite/sleep twin containers from `space-07`: sediment as a physical-space semantic failure.
- Decorative orbit map from `space-08`: sediment as a proof failure because it cannot preserve stable box ownership under phone constraints.

## Synthesis Inputs

- Use `space-04` as the primary synthesis.
- Borrow `space-03` side shelf only as parent context, not as a module picker.
- Borrow `space-05` sleep alcove and wake causality.
- Preserve the baseline mapping regions from `docs/design/mapping/learning-space-implementation-map-v1.md`.
- Reject dashboard counters, twin state boxes, decorative node maps, and card-only drawers without parent context.
