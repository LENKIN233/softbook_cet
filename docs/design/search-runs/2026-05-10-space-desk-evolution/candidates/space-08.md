# DesignCandidate space-08

## Candidate ID

`space-08`

## Provenance

- Tool or model: Codex design search, generation 1
- Prompt: Explore a decorative library globe as a negative control for spatial metaphor.
- Source context pack: docs/design/search-runs/2026-05-10-space-desk-evolution/context-pack.md
- Artifact: Prose candidate record for Library Globe
- Screenshots: No-render rationale: hard-filter rejected before visual proof because decorative orbiting nodes hide stable box ownership and phone containment.

## Product Truth Fit

This candidate uses a spatial metaphor but fails the product truth. Cards orbit library clusters without stable visible ownership. The user may see motion and depth but not the current box and contained cards.

## Focal Object

Decorative globe of orbiting cards, which is not the required current box.

## First-Read Path

Globe center -> orbiting cards -> labels -> current card. The physical hierarchy is delayed and ambiguous.

## Interaction Silhouette

Invalid Space hierarchy silhouette because it uses decorative map physics instead of stable library/group/box/card structure.

## Spatial Model

Ownership is vague. Cards may appear near a library cluster, but the current group and box are not concretely visible.

## State Language

Favorite and sleep would become orbital badges or lanes, which makes state decorative rather than attached to card ownership.

## Motion Causality

Motion would be decorative orbiting. It does not express an allowed Space operation or Learning continuity.

## Platform Strategy

Likely easier on a large canvas than phone. It would become hard to contain at `393 x 852` without tiny labels.

## Implementation Mapping

Would require custom spatial physics and label collision behavior that is not part of accepted implementation mapping.

## Known Risks

Decorative metaphor, hidden ownership, phone label collision, high implementation cost, and weak CET trust.

## Design Review Checklist Answers

Q1: Reading can be colored, but the accent would attach to a decorative cluster rather than the current object.

Q2: Focal object is wrong because the current box is not first-read.

Q3: Space hierarchy silhouette fails because stable box focus is absent.

Q4: Candidate avoids specific forbidden CSS patterns but fails product hierarchy and should not render.

Q5: Phone containment is unlikely because orbit labels and cards would collide.

Q6: Space-only candidate; Learning and stats rules are unchanged, but Space proof fails.
